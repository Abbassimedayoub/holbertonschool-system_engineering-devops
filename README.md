# System Engineering & DevOps

## Overview

Holberton School system design project: **designing web infrastructures** on paper before building them — explaining every component, its role, and the failure modes of each architecture.

## Content

`web_infrastructure_design/` contains four incremental designs:

| Task | Design |
|---|---|
| `0-simple_web_stack` | One server: Nginx, application server, MySQL, DNS — and its single points of failure |
| `1-distributed_web_infrastructure` | Splitting the stack across servers behind a load balancer |
| `2-secured_and_monitored_web_infrastructure` | Firewalls, HTTPS/SSL termination, monitoring agents |
| `3-scale_up` | Component separation and horizontal scaling |

Each file links to a diagram and a written explanation of the architecture.

## Skills Demonstrated

Infrastructure reasoning: load balancing, database replication, TLS, firewalls, monitoring, single-point-of-failure analysis — the vocabulary of DevOps and SRE interviews.
